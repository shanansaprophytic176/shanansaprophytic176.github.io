三、搭建步骤详解

1安装 Hugo Extended 
winget install Hugo.Hugo.Extended

安装后重新打开终端，验证：hugo version

必须安装 Extended 版本，因为主题使用了 SCSS 样式，普通版不支持。
2创建 Hugo 项目 
cd D:\BLOG
hugo new site .

生成的目录结构：

  • content/ - 文章内容（主要编辑这里）
  • config/ - 网站配置
  • themes/ - 主题文件
  • layouts/ - 自定义模板（覆盖主题用）
  • public/ - 构建输出（自动生成，已加入 gitignore）
3安装主题 
git init
git submodule add https://github.com/CaiJimmy/hugo-theme-stack.git themes/hugo-theme-stack

使用 git submodule 安装，方便后续更新主题。

4配置网站

配置文件在 config/_default/ 目录下，详见下方"修改配置"部分。

5撰写文章

在 content/post/ 目录下创建 .md 文件：

---
title: "文章标题"
date: 2026-07-16
draft: false
tags: ["标签1", "标签2"]
categories: ["分类"]
---

这里是文章内容，使用 Markdown 语法编写...
6推送到 GitHub 
git add -A
git commit -m "发布新文章"
git push origin main

推送后 GitHub Actions 自动构建部署，约 1-2 分钟后网站更新。

四、自动部署原理

推送代码后，GitHub Actions 自动执行以下流程：

  1. Checkout - 从 GitHub 拉取最新代码
  2. Install Hugo - 在 Ubuntu 服务器上安装 Hugo Extended
  3. Build - 运行 hugo --gc --minify，把 Markdown 生成 HTML
  4. Deploy - 把生成的静态文件部署到 GitHub Pages
整个过程约 1-2 分钟，完成后网站自动更新，无需手动操作。

七、常见问题

Q: 文章发布后不显示？
A: 检查文章的 draft 是否为 false。如果为 true，文章是草稿状态不会发布。
Q: 网站样式错乱？
A: 删除本地的 public/ 和 resources/_gen/ 目录，重新运行 hugo server -D。
Q: GitHub Actions 部署失败？
A: 打开 GitHub 仓库的 Actions 页面，查看失败的构建日志。常见原因：Hugo 版本不对、配置文件语法错误。
Q: 如何添加新的固定页面（如关于、友链）？
A: 在 content/page/ 下创建新目录，添加 index.md 和 index.zh.md，在 front matter 中添加 menu.main 配置。
Q: 如何修改主题样式？
A: 不要直接修改 themes/ 目录。在 layouts/ 下创建同名文件即可覆盖主题模板。
Q: 如何更新主题？
A: 运行 git submodule update --remote 拉取主题最新版本。
